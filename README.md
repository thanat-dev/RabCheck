# ระบบบันทึกเช็คและรายการจ่ายเงิน (RabCheck)

ระบบสำหรับอัพโหลดรูปเช็คหรือใบโอนเงิน จากมือถือหรือคอมพิวเตอร์ ให้ระบบดึงข้อมูลด้วย OCR (ถ้ามีการติดตั้ง Tesseract) และสามารถกรอก/แก้ไขข้อมูลเองได้ เก็บข้อมูลในฐานข้อมูล และพิมพ์สรุปรายงานได้

## ความสามารถ

- **อัพโหลดรูป**: ถ่ายรูปหรือเลือกรูปจากมือถือ/คอมพิวเตอร์ (หน้าเช็ค, ใบโอนเงิน E-WHT, ใบสรุปรายการ)
- **ดึงข้อมูลอัตโนมัติ**: ใช้ Tesseract OCR (ภาษาไทย+อังกฤษ) ดึงวันที่, จำนวนเงิน, เลขที่เช็ค, ผู้รับเงิน ฯลฯ ถ้ารูปรองรับบาร์โค้ด ระบบจะอ่านมาใส่ "เช็คเลขที่" (pyzbar) และถ้าตั้งค่า API key ระบบใช้ **AI (Google Gemini)** ดูภาพเช็คแล้วดึงเลขที่เช็คออกมาแสดงได้
- **กรอก/แก้ไขเอง**: ทุกรายการแก้ไขได้ในฟอร์ม (วัน/เดือน/ปี, เล่มที่, เลขที่ IV, เช็คเลขที่, จำนวนเงิน, ชื่อผู้ซื้อ/สถานที่ ฯลฯ)
- **เก็บข้อมูล**: เก็บใน SQLite (ไฟล์ `rabcheck.db`)
- **พิมพ์สรุปรายงาน**: ดูสรุปรายการในรูปแบบตาราง และกดพิมพ์ได้

## การติดตั้ง

### 1. Python

ต้องมี Python 3.8 ขึ้นไป

```bash
python --version
```

### 2. ติดตั้งแพ็กเกจ Python

```bash
cd RabCheck
pip install -r requirements.txt
```

### 3. (ถ้าต้องการ OCR ภาษาไทย) ติดตั้ง Tesseract

- **Windows**: ดาวน์โหลดตัวติดตั้งจาก [GitHub tesseract](https://github.com/UB-Mannheim/tesseract/wiki) แล้วติดตั้ง จากนั้นเพิ่มโฟลเดอร์ติดตั้ง (เช่น `C:\Program Files\Tesseract-OCR`) เข้าไปใน PATH และดาวน์โหลดภาษาไทย (tha) ผ่านตัวติดตั้งหรือใส่ไฟล์ `tha.traineddata` ในโฟลเดอร์ `tessdata`
- **Mac**: `brew install tesseract tesseract-lang`
- **Linux**: `sudo apt install tesseract-ocr tesseract-ocr-tha`

ถ้าไม่ติดตั้ง Tesseract ระบบยังใช้ได้ แค่ดึงข้อมูลจากรูปไม่ได้ จะต้องกรอกข้อมูลเองทั้งหมด

### 4. (ถ้าต้องการอ่านบาร์โค้ดเลขที่เช็ค) ติดตั้ง pyzbar

ใน `requirements.txt` มี `pyzbar` อยู่แล้ว — หลัง `pip install -r requirements.txt` ถ้ารูปเช็คมีบาร์โค้ดที่ encode เลขที่เช็ค ระบบจะอ่านมาใส่ใน "เช็คเลขที่" ให้อัตโนมัติ (บน Windows บางเครื่องอาจต้องติดตั้ง [Zbar](https://pypi.org/project/pyzbar/) ตามคู่มือของ pyzbar) ถ้าไม่ติดตั้งหรืออ่านบาร์โค้ดไม่ได้ ระบบยังใช้เลขที่เช็คจาก OCR/MICR ตามเดิม

### 5. (ถ้าต้องการให้ AI ดูภาพแล้วดึงเลขที่เช็ค) Google AI Gemini

ถ้าต้องการให้ระบบใช้ **Google Gemini** วิเคราะห์รูปเช็คแล้วแสดงเลขที่เช็คอัตโนมัติ (ช่วยในเคสที่ OCR อ่านผิด) — ดูรายละเอียดใน **คู่มือติดตั้ง AI Gemini.md**

## การรัน

จากโฟลเดอร์โปรเจกต์ (RabCheck):

```bash
python backend\app.py
```

หรือดับเบิลคลิก `run.bat` (Windows)

จากนั้นเปิดเบราว์เซอร์ที่ **http://localhost:5000**

- **อัพโหลดรูป**: ลากรูปมาวางหรือคลิกเพื่อเลือกรูป
- **รายการทั้งหมด**: ดู/แก้ไข/ลบรายการ หรือกด "เพิ่มรายการ (กรอกเอง)"
- **สรุปรายงาน**: ดูตารางสรุปและกด "พิมพ์สรุปรายงาน"

## โครงสร้างโฟลเดอร์

```
RabCheck/
  backend/
    app.py        # API และรันเซิร์ฟเวอร์
    config.py     # path อัพโหลด, ฐานข้อมูล
    database.py   # SQLite และตาราง
    ocr_helper.py # ดึงข้อความจากรูป (Tesseract)
  frontend/
    index.html
    styles.css
    app.js
  uploads/        # รูปที่อัพโหลด (สร้างอัตโนมัติ)
  rabcheck.db     # ฐานข้อมูล (สร้างอัตโนมัติ)
  requirements.txt
  run.bat
  README.md
```

## หมายเหตุ

- ข้อมูลเก็บในไฟล์ `rabcheck.db` ในโฟลเดอร์โปรเจกต์ ควรสำรองไฟล์นี้เป็นระยะ
- รูปที่อัพโหลดเก็บในโฟลเดอร์ `uploads/`
- การดึงข้อความจากรูปขึ้นกับความชัดของรูปและคุณภาพของ Tesseract ภาษาไทย แนะนำให้ตรวจและแก้ไขในฟอร์มหลังอัพโหลด
