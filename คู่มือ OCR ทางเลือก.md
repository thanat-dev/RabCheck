# คู่มือ OCR ทางเลือกสำหรับ RabCheck

ปัจจุบัน RabCheck ใช้ **Tesseract** แต่หากต้องการความแม่นยำสูงขึ้น มีทางเลือกดังนี้

---

## 1. EasyOCR (แนะนำ — ติดตั้งง่าย)

### ข้อดี
- รองรับภาษาไทย (`th`) + อังกฤษ (`en`) พร้อมกัน
- ตาม benchmark มักแม่นยำกว่า Tesseract สำหรับภาษาไทย
- ติดตั้งง่าย ใช้ `pip install easyocr`
- ไม่ต้องติดตั้งแยก (ต่างจาก Tesseract ที่ต้องติดตั้งอีกโปรแกรม)

### ข้อควรระวัง
- ใช้ PyTorch โปรเจกต์จะใหญ่ขึ้น (~500MB+)
- โหลดโมเดลครั้งแรกใช้เวลาพอสมควร (ดาวน์โหลดอัตโนมัติ)

### การติดตั้ง
```bash
pip install torch torchvision
pip install easyocr
```

### วิธีใช้งานใน RabCheck
1. ติดตั้ง EasyOCR:
   ```bash
   pip install -r requirements-easyocr.txt
   ```
2. ตั้งค่าก่อนรัน (ใช้ Command Prompt):
   ```bash
   set OCR_ENGINE=easyocr
   run.bat
   ```
   หรือแก้ไข `backend/config.py` ให้ `OCR_ENGINE = 'easyocr'`

หมายเหตุ: ยังต้องมี Tesseract ติดตั้งไว้สำหรับการอ่านบรรทัด MICR (digits-only)

## 2. PaddleOCR

### ข้อดี
- แม่นยำสูง รองรับไทย
- มีโมเดล PP-OCRv5 สำหรับหลายภาษา

### ข้อควรระวัง
- ต้องติดตั้ง PaddlePaddle แยก
- โครงสร้างซับซ้อนกว่าหน่อย

### การติดตั้ง
```bash
pip install paddlepaddle
pip install paddleocr
```

---

## 3. Typhoon OCR (เฉพาะภาษาไทย)

### ข้อดี
- ออกแบบมาสำหรับภาษาไทยโดยเฉพาะ
- แก้ปัญหาตัวอักษรไทยได้ดี (สระลอย, ไม่มีช่องว่างระหว่างคำ)
- ใช้ได้ทั้งเอกสารพิมพ์และลายมือ

### ข้อควรระวัง
- โมเดลใหญ่ ต้องการหน่วยความจำมาก
- ติดตั้งและใช้งานซับซ้อนกว่า

### ลิงก์
- https://opentyphoon.ai/
- https://huggingface.co/openthaigpt/thai-trocr

---

## 4. Thai-TrOCR (OpenThaiGPT)

### ข้อดี
- ตาม benchmark ดีกว่า Tesseract ชัดเจน (CER 0.19 vs 1.03 สำหรับลายมือ)
- โอเพนซอร์ส

### ข้อควรระวัง
- ต้องใช้ Hugging Face Transformers
- โมเดลค่อนข้างใหญ่

## ความท้าทายของบรรทัด MICR บนเช็ค

บรรทัด MICR ใช้ฟอนต์พิเศษ **E-13B** ออกแบบเพื่อเครื่องอ่าน ไม่ใช่คน มักทำให้ OCR ทุกตัวอ่านผิดได้:

- ตัวเลข 6/8, 0/O สับสนง่าย
- Tesseract, EasyOCR ต่างก็มีข้อจำกัดกับ MICR

แนวทางที่ดี:
1. ใช้ **OCR digits-only** บนส่วน MICR (มีใน RabCheck แล้ว)
2. ถ้าต้องการแม่นยำขึ้น: พิจารณา **template matching** หรือ **โมเดลเฉพาะ MICR** (มีโซลูชันใน GitHub)
3. เก็บ logic **แก้ไข case-by-case** (ที่เพิ่มไว้แล้ว) เพื่อกรองเลขที่ผิด

---

## ข้อแนะนำ

- **ถ้าต้องการเปลี่ยนจาก Tesseract:** ลอง **EasyOCR** ก่อน เพราะติดตั้งง่ายและมักแม่นยำกว่าสำหรับภาษาไทย
- **ถ้าต้องการความแม่นยำสูงสุด:** พิจารณา **Typhoon OCR** หรือ **Thai-TrOCR**
- **สำหรับเลขเช็ค (MICR):** ยังต้องพึ่ง logic แก้ไขใน RabCheck เพราะ MICR เป็นฟอนต์พิเศษ
